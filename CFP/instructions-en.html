Guidance for writing grants and contribution funding pages

From: CSB Portfolio Web | Data and Optimization Team

Funding pages focus on helping people complete tasks associated with applying for funding for a specific program. This is an Employment and Social Development (ESDC) specific template created for contribution grants and funding pages based on user research and testing with users.

These pages help people:

  • determine eligibility for funding program
  • learn what information is needed to complete the application
  • learn how to submit the application
  • learn what happens after the application is submitted
  • find resources and contact information for help
  • Benefits of this template

    While there are many ways this information can be presented, this template should be used for all call for proposals (CFP) for grants and contributions (Gs & Cs) funding programs.

    This template:

    • supports a consistent approach to task initiation and completion, including ordering by steps
    • provides people with the information they need before initiating the task
    • offers alternate methods to apply (for example: online, in-person or by mail)
    • ensures that experienced applicants can quickly initiate a task and choose whether to review the supporting information offered
    • breaks up long and complex content into pages that each focus on a step or specific answer that people need before moving to the next step or section.

When to use this template

  • Use this template for all CFP processes for funding of grants and contribution (Gs & Cs) programs.

What to avoid

  • Don't use numbers when you use this pattern for sequenced steps. Testing has shown that Canadians are less likely to get to the section they need if the elements are numbered.
  • Avoid using sub-steps within this pattern, as we haven't yet done testing for this variation.

How to implement

The purpose of this template is to provide a common structure to the Funding Page, including the Application Instructions, to standardize language across programs. The Grants and Contribution Funding Page Template is used to help applicants apply for funding under the:

  • Standard Contribution Application Form (EMP5523) or
  • Standard Grant Application for Funding (EMP5671)

This template also applies to programs that have specific forms for their application for funding.

Should your guide differ from this template, ADM approval is required.

Guidance for template pages

In this new template, there are 5 pages described in the following dropdown sections.

Each page contains sections. Each section will contain content that is either program-specific, generic, or customizable.

Program-specific
Used for sections where program-specific information is required or text needs to be customized.
Generic
Used for sections where generic information is included. Use that text unless there is a good reason to use something else.
Customizable
Used for sections where you may need to customize the generic text.
Mandatory
Used for rules (not guidance) that must be followed and information that must be included in the web page.

Template page and sections guidance

0. Index page
Page purpose

Provides a general overview of the funding program and gives key details at a glance.

Working example of index page

Page sections
Page title Customize Mandatory

The name of the program. If there are multiple streams or components, include the stream title.

Funding program summary Customize Mandatory

A short description of the program in no more than 200 words with enough details to provide a potential applicant with sufficient information to determine if this program is of interest to their organization.

How to apply guide Generic Mandatory

Links to each section of the applicant guide with a brief description of what each page contains.

Information sessions Customize

Provides applicants with the dates and times of the information session and how to sign up to attend.

Contact Customize Mandatory

Contact information and direct links to answers to frequently asked questions.

The following is mandatory in this section:

  • General information (if your program has a separate contact page, you can link to it instead)
  • Accessibility and disability help

All other sections are optional.

FAQ (optional)

The FAQ section is optional. Ideally, we hope that the information is organized and written clearly so that people can find the answers to their questions within the content. However, when an application period is open, that is not the time to re-write and re-design content. To help users during this period, we can add this FAQ section. Add a link to the section of the CFP that addresses the question. This avoids duplicating content.

Page purpose

Working example of the "What this program offers" page

Page sections
Program description Program-specific Mandatory

In one or two paragraphs, describe the funding. Outline the amount of funding available and include the maximum amount of funding per agreement, if applicable. If there are other streams of the same program open at the same time, provide links to these streams, as well.

Keep this section short. Focus on the details relevant to an applicant applying for this specific funding stream. Place details on the official program description page. Add a link to that page in this section if needed.

2. "Who can apply" page
Page purpose

Describes the eligibility criteria for an applicant and how the application is assessed.

Working example of "Who can apply" page

Page sections
In-page table of contents Program-specific Mandatory

The in-page table of contents pattern is a list of links that lead to subsections of the same page. This pattern is intended for long documents that exist as a single page.

In-page table of contents

Eligibility Program-specific Mandatory

Applicants Program-specific

Describe the eligibility criteria for an applicant. For example, specify who can apply. List the eligible recipients, as applicable.

Projects Program-specific

Describe the eligibility criteria for projects. For example, specify what type of projects are acceptable. Indicate the maximum duration and maximum funding per project. List the eligible projects, as applicable, and provide examples, if possible.

Activities Program-specific

Describe the eligibility criteria for activities. For example, specify what type of activities are acceptable. List the eligible activities, as applicable, and provide examples, if possible.

Who can participate in your project Program-specific

If applicable to your program, describe in this section the eligibility criteria for participants. For example, specify who can be a participant on a project.

What the funding can be used to pay for Program-specific

Describe the eligibility criteria for costs/expenditures. For example, specify what an applicant can spend money on or purchase. List the eligible costs, as applicable, and provide examples, if possible.

Other eligibility requirements Program-specific

Include any other eligibility requirements in this section. This would include such things as the application being received by the deadline (clearly indicate the deadline), the application being completed in full and “signed”, the attestation being checked, and required supporting documents being submitted.

How your application is assessed Program-specific

Outline how submitted applications are assessed. Include the details in expand/collapse components.

Accessibility

Note that your CFP should be accessible and written in plain language. The readability requirements for web pages is between a grade 6 to 8 level.

Follow the accessibility guidelines provided by Shared Services Canada.

Official languages

As a federal institution, ESDC is subject to the Official Languages Act (OLA).

Official Languages laws and regulations
  • Communications with and Services to the Public | Part IV: Information about the language employees use when serving citizens, languages to be used in communications to the public, services of equal quality, active offers of service, etc.
  • Language of Work, Equitable Participation and Staffing Generally | Parts V, VI and Section 91: Information about language rights at work as a federal government employee, official language obligations as a manager, equal opportunities for employment and advancement for English and French speaking Canadians, determination of language requirements of positions, etc.
  • Advancement of English and French | Part VII: Information about official language minority communities, programs and policies that meet the needs of these communities, fostering the full recognition and use of both official languages, etc.

Under the OLA, every department has the duty to take positive measures to implement Canada’s commitment to enhancing the vitality of Official Languages (OL). An example of positive measure is to make official language minority communities (OLMCs) a priority for an intake process. Please refer to the Official Languages LENS to learn more.

ESDC must inform applicants in advance if it intends to prioritize projects national in scope catering to both OL. This will allow them to develop their application accordingly. Also, ESDC must indicate if the intent is to only fund projects national in scope that are bilingual. The documents must identify this as a requirement to inform applicants before they apply.

It is not mandatory for all projects national in scope to offer bilingual services. The program's Terms and Conditions (Ts&Cs) would need to specify this eligibility criteria (Treasury Board [TB] Submission stage). However, the program may set requirements to ensure services and programs are available to both OL communities. Please refer to the Program Design and Delivery Planning Framework. It offers OL considerations for the Memorandum to Cabinet and TB Submission stages.
