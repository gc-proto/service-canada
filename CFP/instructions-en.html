Contribution Funding Page Guidance

From: CSB Portfolio Web

Contribution funding pages focus on helping people complete tasks associated with applying for funding for a specific program. This is an ESDC specific template created for contribution funding pages based on user reasearch and testing with users.

These pages help people:

  • determine eligibility for funding program
  • learn what information is needed to complete the application
  • learn how to submit the application
  • learn what happens after the application is submitted
  • find resources and contact information for help
  • Benefits of this template

    While there are many ways this information can be presented, this template should be used for all contributions funding programs.

    This template:

    • supports a consistent approach to task initiation and completion, including ordering by steps
    • provides people with the information they need before initiating the task
    • offers alternate methods to apply (i.e. online, in-person or by mail)
    • ensures that experienced clients can quickly initiate a task and choose whether or not to review all of the supporting information offered
    • breaks up long and complex content into pages that each focus on a step or specific answer people need before moving to the next step or section.

When to use this template

  • Use this template for unsolicited contribution funding program streams.

What to avoid

  • Don't use numbers when you use this pattern for sequenced steps. Testing has shown that Canadians are less likely to get to the section they need if the elements are numbered.
  • Avoid using sub-steps within this pattern, as we haven't yet done testing for this variation.

How to implement

The purpose of this document is to provide a common structure to the Funding Page, including the Application Instructions, written for Contribution programs under Employment and Social Development Canada (ESDC) in order to standardize language across programs. The Contribution Funding Page Template is used to help applicants apply for funding under the Standard Contribution Application Form (EMP5523).

Should your guide differ from this template, ADM approval is required.

Template pages guidance

In this new template, there are 5 pages described in the following dropdown sections.

Section label definitions

Each page contains sections. Each section will contain content that is either program specific, generic, or customizable.

Program specific

Used for sections where program-specific information is required or text needs to be customized.

Generic

Used for sections where generic information is included. Use that text unless there is a good reason to use something else.

Customizable

Used for sections where you may need to customize the generic text.

Mandatory

Used for rules (not guidance) that must be followed.

Template page and sections guidance

  • 1. Index page
    Page purpose

    Provides a general overview of the funding program and gives key details at a glance.

    Working example of index page

    Page sections
    Page title Customize

    The name of the program. If there are multiple streams or componenents, include the stream title.

    Funding program summary Customize

    A short description of the program in no more than 200 words and enough details to provide a potential applicant with enough information to determine if this program is of interest to their organization.

    Application Guide Generic

    Links to each section of the applicant guide with a brief description of what each page contains.

    Information Sessions Customize

    Provides applicants with the dates and times of the information session and how to sign up to attend.

    Contact Customize

    Contact information and direct links to answers to frequently asked questions.

    The following is mandatory in this section:

    • General information
    • Accessibility and disability help

    All other sections are optional.

    FAQ

    The FAQ section is optional. Ideally, we hope that the information is organized and written clearly so that people can find the answers to their questions within the content. However, when an application period is open, that is not the time to re-write and re-design content. To help users during this period, we can this FAQ section. Add a link to the section of the CFP that addresses the question. This avoids duplicating content.

    Information architecture and navigation Mandatory

    The following shows where content should be saved and what the breadcrumbs should look like on the CFP page:

    Home → Theme → Topic and/or sub-topic → [funding program index page]

    The following shows areas where your CFP page should be linked from:

    Home → Employment and Social Development Canada → Funding programs → [funding program index page]


    Home → Employment and Social Development Canada → Programs → [funding program index page]

  • 2. What this program offers page
    Page purpose

    Describes the funding program and it’s objectives.

    Working example of what this program offers page

    Page sections
    Program description Program specific

    In one or two paragraphs, describe the funding. Outline the amount of funding available and include the maximum amount of funding per agreement, if applicable. If there are other streams of the same program open at the same time, provide links to these streams, as well.

    Keep this section short. Focus on the details relevant to an applicant applying for this specific funding stream. Place details on the official program description page. Add a link to that page in this section if needed.

    In-page table of contents Program specific

    The in-page table of contents pattern is a list of links that lead to subsections of the same page. This pattern is intended for long documents that exist as a single page.

    In-page table of contents

    Program objectives Program specific

    Using bullet points describe the objectives of the program. If there are National, Regional or Local priorities for this intake, list these, as well.

    Funding priorities Program specific

    If priority may be given to projects meeting certain criteria, list these in this section.

    Glossary Customize

    List of key terms used throughout the application guide. Use the definition lists design pattern for this section. Remove the definitions you do not need. Add definitions if they are not already on this list.

    Definition lists

    3. Eligibility and assessment page
    Page purpose

    Describes the eligibility criteria for an applicant and how the application is assessed.

    Working example of eligibility and assessment page

    Page sections
    In-page table of contents Program specific

    The in-page table of contents pattern is a list of links that lead to subsections of the same page. This pattern is intended for long documents that exist as a single page.

    In-page table of contents

    Who can apply Program specific

    Describe the eligibility criteria for an applicant, i.e. who can apply. List the eligible recipients, as applicable.

    Eligible projects Program specific

    Describe the eligibility criteria for projects, i.e. what type of projects are acceptable. Indicate the maximum duration and maximum funding per project. List the eligible projects, as applicable, and provide examples, if possible.

    Eligible activities Program specific

    Describe the eligibility criteria for activities, i.e. what type of activities are acceptable. List the eligible activities, as applicable, and provide examples, if possible.

    Who can participate in your project Program specific

    If applicable to your program, in this section describe the eligibility criteria for participants, i.e. who is able to be a participant on a project.

    What the funding can be used to pay for Program specific

    Describe the eligibility criteria for costs/expenditures, i.e. what can an applicant spend money on or purchase. List the eligible costs, as applicable, and provide examples, if possible.

    Other eligibility requirements Program specific

    Include any other eligibility requirements in this section. This would include such things as the application being received by the deadline (clearly indicate the deadline), the application being completed in full and “signed”, the attestation being checked, required supporting documents being submitted, etc.

    How your application is assessed Program specific

    Outline how submitted applications are assessed. Include the details in expand/collapses.

    5. How to apply page
    Page purpose

    Overview of how to apply and what happens after you apply.

    Working example of how to apply page

    Page sections
    Application period Program specific

    The first sentence on this page repeats the application period dates.

    Apply now and other ways to applyCustomize

    Advise the applicant how to submit their application for your program. Indicate if applications are accepted via GCOS, Online fillable form (IFFS), Email, Fax, Mail, or In-person. Ensure the preferred way to apply comes first on the list. The web team will put the other ways to apply below this in a dropdown menu.

    Submit one application Generic

    Remind applicants to submit their application once and not multiple times.

    Funding decision Customize

    This section is mostly program-specific, but does include some text that will be generic to all programs. The purpose of this section is to provide information to applicants on what happens after the application has been submitted.
